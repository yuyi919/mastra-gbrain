# Quick Task 260425-q02 Summary

## Outcome

为仓库接入了 Husky 与 lint-staged 的 pre-commit 规范，提交前会通过 `pnpm lint-staged` 触发 `check:fix`。`check:fix` 现在通过 `scripts/check-fix.mjs` 调用 Biome，无参数时检查整个仓库，有文件参数时只处理传入的 staged 文件。

## Delivered Files

- `package.json`
- `pnpm-lock.yaml`
- `scripts/check-fix.mjs`
- `.husky/.gitignore`
- `.husky/pre-commit`

## Key Decisions

- 保留 `check:fix` 作为唯一修复入口，而不是让 hook 直接写死 Biome 命令。
- 用一个小包装脚本兼容两种调用方式：
  - 手动执行 `pnpm check:fix` 时扫全仓
  - lint-staged 传文件参数时只处理 staged 文件
- 通过 `prepare` 自动执行 Husky 安装，避免每次手工重装 hook。
- 通过 `.husky/.gitignore` 排除 Husky 安装生成的 `./.husky/_` 辅助文件，避免把本地生成物提交进仓库。
