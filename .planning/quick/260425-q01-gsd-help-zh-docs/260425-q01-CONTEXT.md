# Quick Task 260425-q01: GSD 中文使用文档整理 - Context

**Gathered:** 2026-04-25
**Status:** Ready for planning

<domain>
## Task Boundary

将 `$gsd-help` 提供的命令参考整理成项目内的中文使用文档，产物放在 `docs/` 下，可以拆成多个文件。

文档目标不是逐条解释每个命令，而是按常用工作模式组织内容，让读者先理解 GSD 的项目节奏，再知道在不同情境下该跑哪条命令链。

</domain>

<decisions>
## Implementation Decisions

### 文档组织方式
- 采用多文件结构，而不是单一大文件。
- 至少拆成总览、常用流程、异常处置、文档生命周期四部分。

### 内容重点
- 重点说明几个常用模式的完整命令流程。
- 强调里程碑、阶段、quick task 之间的区别和使用边界。
- 必须单独回答“里程碑完成后散落文档如何处置”。

### 表达方式
- 全文中文。
- 面向已经在本仓库里使用 GSD 的开发者，尽量直接、可执行。
- 不按命令字典展开，不把帮助文档机械翻译成一串命令注释。

### the agent's Discretion
- 具体文件名与章节切分。
- 每个流程中列哪些异常场景，只要覆盖最常见的跑偏情形即可。

</decisions>

<specifics>
## Specific Ideas

- 用“模式一到模式四”覆盖最常见的主流程。
- 单独写文档生命周期，明确区分常驻文档、当前工作文档、历史归档文档。

</specifics>
