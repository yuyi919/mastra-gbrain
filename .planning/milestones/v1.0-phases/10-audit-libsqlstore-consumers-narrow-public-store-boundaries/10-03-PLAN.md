<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| branch services to vector store | Retrieval, lifecycle, and maintenance code issue vector operations through an internal provider. |
| public facade to internal vector provider | `LibSQLStore.vectorStore` remains public compatibility, but internal branch code should not depend on that raw object. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | Tampering | Vector branch dependency injection | mitigate | Replace raw `LibSQLVector` dependency passing with `VectorProviderService`, and test fake provider calls. |
| T-10-08 | Information Disclosure | `OpsInternalService` | mitigate | Remove raw vector client exposure from branch internal service; keep public raw facade only on `LibSQLStore`. |
| T-10-09 | Denial of Service | vector provider lifecycle | mitigate | Provider owns `createIndex` and `dispose` so lifecycle cleanup stays centralized and testable. |
| T-10-10 | Elevation of Privilege | `libsql-store.ts` assembly | mitigate | `libsql-store.ts` composes layers only and must not reimplement branch vector operations. |
</threat_model>
