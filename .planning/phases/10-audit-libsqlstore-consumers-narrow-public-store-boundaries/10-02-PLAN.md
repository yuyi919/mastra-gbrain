<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workflow caller to ingestion workflow | Public and tool callers pass an injected store and embedder into the workflow. |
| injected store to persistence operations | The workflow writes pages, tags, chunks, and timeline entries through the store dependency. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04 | Elevation of Privilege | `IngestionOptions.store` | mitigate | Replace `StoreProvider` with `IngestionWorkflowStore` so workflow code cannot access raw data, config, search, lifecycle, or unsafe helper methods. |
| T-10-05 | Tampering | `transaction` callback | mitigate | Type transaction callback as `(tx: IngestionWorkflowStore)` so transactions do not reintroduce broad public facade access. |
| T-10-06 | Information Disclosure | workflow/provider surface | mitigate | Block imports of `BrainStoreTree`, branch services, or raw store internals in workflow callers. |
</threat_model>
