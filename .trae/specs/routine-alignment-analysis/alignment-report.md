# 核心功能对齐分析报告

## 概述
这是一份关于 `/tmp/gbrain` (参考实现) 与本地 `/workspace/src/` (当前实现) 核心功能的详细对比报告。
本地代码库在整体架构上引入了 Mastra Workflows (用于 import) 和 SQLite/LibSQL (用于存储)，但在具体业务逻辑和安全防护上，遗漏了大量原版 gbrain 积累的防御性编程逻辑和边缘用例处理。

---

## 1. `searchKeyword` 去重逻辑
### 结构逻辑差异
- **GBrain**: 采用精密的 **4层去重流水线**：
  1. 按来源：每个页面最多保留得分最高的 3 个 chunk。
  2. 文本相似度：基于 Jaccard 相似度（>0.85）剔除语义重复的 chunk。
  3. 类型多样性：限制单一页面类型（如全都是 people 或 meeting）在结果中占比不超过 60%。
  4. 最终硬性限制：每页最多保留 2 个 chunk。
  5. **Compiled Truth 保证**：最终检查并确保每个页面至少包含 1 个 `compiled_truth`（核心摘要）类型的 chunk。
- **本地**: 逻辑极其简单，仅通过判断 `slug` + `chunk_text` 的前50个字符是否绝对一致来进行 `Set` 过滤。

### 本地遗漏的边缘用例与风险
- **语义重复污染**：缺少文本相似度过滤，同一页面中措辞略有不同但语义相同的文本会重复出现在搜索结果中。
- **信息不全**：可能只返回零碎切片，而遗漏该页面最重要的核心总结 (`compiled_truth`)。
- **结果同质化**：缺少单页 Chunk 数量上限和全局类型多样性限制，一次搜索可能会被单一长文档或单一类型数据“刷屏”。

---

## 2. `check-backlinks` (反向链接维护)
### 结构逻辑差异
- **GBrain**: 针对实体（`people`, `companies`）进行提取。生成反向链接时，会读取源文件的 Title，并精准计算目标文件到源文件的相对路径层级（通过 `../` 自动推导深度），最后将其安全插入到 `## Timeline` 标题下方。
- **本地**: 基于 AST (`remark`) 提取链接。回写时，直接提取文件名作为 Link 标题，并在文件末尾追加。

### 本地遗漏的边缘用例与风险
- **路径计算灾难 (Critical Bug)**：本地写死了相对路径格式 `./${basename(s)}`。如果源文件和目标文件不在同一个目录下，生成的反向链接点击后将全部报 404。
- **排版破坏**：本地没有识别或维护 `## Timeline` 的逻辑，仅仅在底部粗暴追加，容易破坏 Markdown 文档原有的结构语义。
- **缺乏安全预览**：本地缺少 `--dry-run` 参数，在批量修改文件时存在不可逆的风险。

---

## 3. `doctor` (系统健康检查)
### 结构逻辑差异
- **GBrain**: 深度检查数据库的 Schema 版本 (`LATEST_VERSION`)。支持 `--json` 输出。
- **本地**: 增加了 FTS5 虚拟全文搜索表的完整性校验，但缺少版本对齐和自动化输出支持。

### 本地遗漏的边缘用例与风险
- **缺乏版本对齐检查**：没有校验当前的 Schema 版本，如果用户忘记执行数据库迁移，无法给出明确警告。
- **CI/CD 友好度**：缺少 `--json` 输出支持，外部脚本或 Agent 难以通过解析 JSON 来自动化判断系统健康度。

---

## 4. `import` (本地文件导入)
### 结构逻辑差异
- **GBrain**: 包含多线程 Worker 池并发导入；每 100 个文件写入一次检查点支持断点续传；主动拦截软链接；跳过隐藏文件和 `node_modules`。
- **本地**: 逻辑解耦更清晰（基于 Mastra），但目录遍历仅使用了简单的单线程递归。

### 本地遗漏的边缘用例与风险
- **安全漏洞 (Security Risk)**：直接读取软链接，存在严重的安全隐患（可能被利用读取系统敏感文件）。
- **垃圾目录过滤缺失**：未跳过隐藏目录和 `node_modules`，极易导致摄入大量垃圾文件。
- **崩溃无法恢复**：缺乏断点续传机制，一旦中断只能全部重新扫描。
- **性能瓶颈**：单线程递归速度慢，处理大规模文档库时效率低下。
